Przygotuj agenta, który ułatwi mi implementacje aplikacji i zmian, na serwerze developerskim.

Server docker
hostname: debian
ip: 192.168.0.106

źródło aplikacji repozytoria github: https://github.com/fatalny-bit/baza-ksiazek.git

Co jakie informacje będą potrzebne, by po wypushowaniu zmian na github, zmiany w kodzie były implementowane na serwerze.


To do:

Potrzebujesz:
V użytkownika (np. devel)
V IP: 192.168.0.106
V port SSH (domyślnie 22)
V klucz prywatny (dodany do GitHub Secrets)
V klucz publiczny (dodany do ~/.ssh/authorized_keys na serwerze)

Lokalizacja aplikacji na serwerze
Musisz ustalić:
V katalog, w którym będzie stał projekt, np. /srv/baza-ksiazek
plik docker-compose.yml w tym katalogu
skrypt deployujący, np. /usr/local/bin/deploy.sh
Agent musi wiedzieć, gdzie ma wykonać docker compose pull i docker compose up -d.


Rejestr obrazów Dockera
GitHub Actions musi gdzieś wypchnąć zbudowany obraz.
Opcje:
Docker Hub
GitHub Container Registry (GHCR)
prywatny registry na Twoim serwerze
Potrzebne dane:
adres registry
nazwa obrazu (np. ghcr.io/fatalny-bit/baza-ksiazek:latest)
token dostępu (w GitHub Secrets)

Musisz określić:
na jaki branch reagujemy (np. main lub dev)
jak budować obraz Dockera
jak wypychać obraz do registry
jak po SSH wywołać skrypt deployujący
To jest serce agenta CI/CD.
